# 文档评估与自动提交

请执行以下自动化工作流：

## 步骤 1：读取评估指南

读取 `docs/v1/guides/dev-文档评估指南.md`，找到第一个状态为 `⬜` 的待评估文档。

## 步骤 2：读取 SSOT

读取 `docs/v1/design/状态枚举定义.md` 作为状态值的唯一标准。

## 步骤 3：执行评估

对目标文档进行系统性评估，检查：

1. **状态枚举一致性**
   - bind_status、pay_status、camp_status、bind_method、accessToken状态
   - 与 SSOT 定义是否一致

2. **已废弃概念**
   - 智能匹配 (smart_match)
   - 匹配置信度
   - FastAuth方案
   - dynamic_qrcode

3. **文档链接**
   - 相关文���链接是否有效
   - 是否引用了已归档的文档

4. **状态机图**
   - 状态转换是否与 SSOT 一致

## 步骤 4：修复问题

直接编辑目标文档，修复发现的问题。

## 步骤 5：更新评估指南

更新 `docs/v1/guides/dev-文档评估指南.md`：
- 将该文档状态改为 `✅ 已评估并修复`
- 在"发现的问题"表格添加记录
- 在"修复记录"表格添加记录
- 更新"评估进度"统计

## 步骤 6：提交并推送

执行 Git 操作：
```bash
git add .
git commit -m "docs(v1): 评估并修复[文档名]，统一状态枚举"
git push origin main
```

## 步骤 7：输出摘要

输出评估摘要，包括：
- 评估的文档
- 发现的问题数
- 修复的问题数
- 下一个待评估文档

---

**提示**：评估完成后，用户可以运行 `/compact` 压缩上下文，然后再次运行 `/evaluate-next` 继续下一个文档。
